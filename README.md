# extract_rooms_v2

Сервис для администраторов студсовета ННГУ: автоматизация подбора свободных аудиторий и генерация служебных записок (PDF) через Telegram-бота.

## Возможности
- **Интерактивный Telegram-бот:** Многорежимный интерфейс для поиска свободных аудиторий и формирования документов.
- **Генерация служебных записок (PDF):** Автоматическое создание файлов по встроенным шаблонам (Расписание, Холл, Освобождение, Внос-Вынос).
- **Подбор аудиторий:** Умный поиск помещений для пакета запросов без пересечений внутри одного батча. Поддержка различных типов вместимости (`any`, `big`, `big2`, конкретный номер и т.д.).
- **Фоновое обновление данных:** Скачивание raw JSON по корпусам из `/ruzapi/schedule/building/{building_oid}`, обрезка до минимального формата и кэширование в in-memory репозитории.
- **CLI-инструменты:** Различные режимы запуска для тестирования аллокации, генерации PDF и загрузки расписания без запуска бота.

## Формат входного запроса (для поиска аудиторий)
В режиме планирования бот и CLI принимают строки в следующем формате:
`Имя Фамилия Цель дд.мм чч:мм чч:мм тип_аудитории`

Поддерживаемые типы аудитории:
- `any`, `any2`, `any6`
- `big`, `big2`, `big6`
- Конкретный номер аудитории (например, `140` или `105 (корпус 2)`)

## Требования и настройка
Убедитесь, что у вас настроены необходимые конфигурационные файлы и переменные окружения:

1. Скопируйте шаблон конфигурации:
   ```bash
   cp config.example.json config.json
   ```
2. Для работы генератора PDF убедитесь, что в директории `fonts/` лежит нужный шрифт (например, `times.ttf`), а также присутствует файл `pdf_rooms.config`.
3. Установите токен вашего Telegram-бота в переменную окружения `TELEGRAM_BOT_TOKEN`

## Запуск
Основной режим работы (запускает одновременно Telegram-бота и планировщик обновления расписания):
```bash
python -m app.main run --config config.json
```

### Вспомогательные CLI-режимы
Инструмент поддерживает дополнительные команды для ручного управления и отладки:

- **Принудительное обновление расписания:**
  ```bash
  python -m app.main refresh --config config.json
  ```
- **Запуск только планировщика (без бота):**
  ```bash
  python -m app.main scheduler --config config.json
  ```
- **Локальное тестирование подбора аудитории:**
  ```bash
  python -m app.main allocate --config config.json --request "Иван Иванов Консультация 12.02 09:00 10:30 any"
  ```
- **Тестирование генерации PDF:**
  ```bash
  python -m app.main pdf --config config.json --request "Иван Иванов Консультация 12.02 09:00 10:30 any" --pdf-path "output/report.pdf"
  ```

## Примечания
- Если нужной даты нет в загруженном расписании, сервис вернет статус: `no day in schedule`.
- Если под заданные критерии и время нет свободных помещений: `no free room`.
- Логи работы сервиса и ошибки генерации выводятся в стандартный поток вывода (`stdout`).
